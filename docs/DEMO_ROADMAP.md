# LowAir-GS 演示路线图 (Demo Roadmap)

为了保证项目的工程化落地质量，LowAir-GS 采取“小步快跑、边界清晰”的迭代策略。本项目规划了 5 个核心 Demo，每个 Demo 都有严格的边界限制和明确的验收标准。

---

## Demo-01: 基础框架与局部坐标系验证 (已完成)

**核心目标：** 跑通 Qt + OpenGL 渲染管线，验证 UDP 遥测接收与 UI 状态更新机制。

**功能边界：**
- [x] **场景来源**：程序内部硬编码的简化低空场景（网格、坐标轴、建筑块、起降点、参考航线）。
- [x] **坐标系统**：使用纯局部笛卡尔坐标系（SCENE坐标），不做任何真实地理坐标转换。
- [x] **数据来源**：Python 模拟器通过 UDP JSON 发送模拟遥测数据。
- [x] **外部依赖**：**零外部三维库依赖**（仅依赖 Qt6），无需 Assimp、PCL、OSG 等。

---

## Demo-02: 静态三维资产加载与渲染 (规划中)

**核心目标：** 引入第三方模型加载库，实现外部静态三维资产（摄影测量网格）的加载与渲染。

**功能边界：**
- [ ] **场景来源**：外部 OBJ / PLY / GLB 格式的摄影测量或手工建模资产。
- [ ] **外部依赖**：引入 Assimp 或 TinyGLTF 等模型解析库。
- [ ] **渲染能力**：实现基础的光照、材质解析和纹理贴图（Texture Mapping）。
- [ ] **限制**：仍使用局部坐标系，模型位置通过手动偏移（Offset）对齐。

---

## Demo-03: 真实地理坐标系系转换 (规划中)

**核心目标：** 解决真实世界坐标与 OpenGL 局部渲染坐标的对齐问题。

**功能边界：**
- [ ] **坐标系统**：实现 WGS84（经纬高） → ENU（东北天） → SCENE（OpenGL局部）的完整转换链路。
- [ ] **外部依赖**：引入 GeographicLib 或 PROJ 库处理地球椭球体投影。
- [ ] **数据来源**：Python 模拟器发送带有真实经纬度的遥测数据。
- [ ] **限制**：不接入真实飞行器，仅处理坐标数学转换。

---

## Demo-04: 标准协议与真实无人机接入 (规划中)

**核心目标：** 替换自定义的 UDP JSON 协议，接入工业界标准无人机通信协议。

**功能边界：**
- [ ] **协议支持**：实现 MAVLink 协议解析（如接入 PX4/ArduPilot 飞控）或 ROS2 节点通信。
- [ ] **数据来源**：SITL（软件在环仿真）或真实无人机硬件。
- [ ] **外部依赖**：引入 MAVSDK-C++ 或 rclcpp。
- [ ] **限制**：仅做遥测数据下行（Telemetry），不做上行控制（Control）。

---

## Demo-05: 3D Gaussian Splatting 渲染集成 (终极目标)

**核心目标：** 突破传统多边形网格（Mesh）渲染限制，实现前沿的 3DGS 神经辐射场实时渲染。

**功能边界：**
- [ ] **场景来源**：由 COLMAP / 预训练模型生成的 `.splat` 或 `.ply` 格式的 3DGS 资产。
- [ ] **渲染能力**：集成基于 CUDA / Vulkan / OpenGL Compute Shader 的 3DGS 光栅化器。
- [ ] **性能指标**：在消费级 GPU 上实现 1080P 分辨率 30fps 以上的渲染帧率。
- [ ] **虚实融合**：将传统多边形（无人机、轨迹）与 3DGS 体积渲染结果进行深度混合（Depth Compositing）。
